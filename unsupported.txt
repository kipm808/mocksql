MOCKSQL UNSUPPORTED FEATURES
============================

This document lists SQL Server features that are NOT supported by mocksql.
mocksql is designed for testing basic SQL operations, not as a complete
SQL Server replacement.

Last Updated: 2026-05-25

NOTES
=====

1. mocksql is designed for TESTING, not production use
2. Focus is on core SQL DML/DDL operations for integration tests
3. No schema enforcement (constraints, foreign keys, etc.)
4. No query optimization (full table scans only)
5. Authentication accepts any credentials
6. Authorization (GRANT/REVOKE) is parsed but not enforced
7. Transactions are parsed but ACID guarantees are file-based only
8. System views (sys.tables, sys.columns) return minimal data


WHAT IS SUPPORTED
=================

See README.md for the full list of 460+ tested features including:
- Core SELECT/INSERT/UPDATE/DELETE
- JOINs (INNER, LEFT, RIGHT, FULL, CROSS)
- Subqueries and CTEs
- Aggregation (GROUP BY, HAVING)
- Set operations (UNION, INTERSECT, EXCEPT)
- CASE expressions
- WHERE clause with all operators
- ORDER BY and pagination (TOP, OFFSET-FETCH)
- Basic DDL (CREATE TABLE, DROP TABLE, TRUNCATE)
- Parameterized queries (sp_executesql)
- NULL handling and three-valued logic


For detailed supported features, run the test suites:
  cd mocksql && cargo test --test integration
  cd sqltests && cargo run
